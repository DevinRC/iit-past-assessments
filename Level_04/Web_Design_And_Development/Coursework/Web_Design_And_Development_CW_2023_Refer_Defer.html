Specification:

  • Implementation (72 marks)
    Prepare a Web site about yourself. This should include the following.
    1. A presentation page with title. After 3 seconds, a second page will be displayed (Main page).
    2. Your Main page should contain navigation links to the other pages of your website and the links should look like buttons. The 'buttons' should have a hover effect and be created using CSS and HTML. Your site should contain a separate page for each of these features: your CV, ideal job, query form and SVG diagram (see details below).
    3. Your CV with headings and anchors (e.g. links at the top to take the user to various sections of your CV such as Education, Work Experience, etc )
    4. Your ideal job (description, location, responsibilities, colleagues, etc.)
    5. A form to allow people to send you a query about your CV or your ideal job. ( For example, a query about your programing experience, other experience, transferable skills etc). The form should be set up to email the contents of the form to yourselves, but this will take 2 steps, see details below. This form should include:
      • input fields for users to enter their details (name, email address)
      • a textarea to enter their query detail
      • a way to select the theme of the query (using radio buttons or select). There must be a minimum of 3 subjects to choose from (e.g., programing experience, other experience, transferable skills).
      • a 'view query' button. When this button is clicked, check, using JavaScript, that the user has filled-in the compulsory fields - 'name', 'email address', 'query subject' and 'query details'. Do NOT use HTML5 validation for this part. If some fields are missing, use a popup message (alert) to let the user know which fields are missing.
      • If all the required fields are filled in, the name, email address, query subject, and the query detail from the form should be displayed on the page (using innerHTML) in a summary that will include all these details (the name, email address, query subject and the query details).
      • For example, if the user entered 'Juan' for their name, 'juan@gmail.com' for their email address, and the subject of the query was 'programming experience' and they entered 'How much experience do you have on JavaScript?' as the text of their query detail, when they press the check button, the form disappears and the display on the page should say something like:

        Name: Juan
        Email: juan@gmail.com
        Subject: programming experience
        Details: How much experience do you have on JavaScript?

        With two new buttons: Edit and Send.

      • When the user presses edit, the displayed details disappear and this is replaced with the initial form which reappears with the information filled in. If they press send, the displayed details disappear also and the form is sent by email.
    6. Your site should contain an interactive SVG diagram to show the architecture of your site.
    7. Consistency of style should be used throughout the Web site, with the use a common external style sheet file.
    8. Each page should include your title logo at the top, and a link your email address at the bottom of the page.
    9. Feel free to use Tables, Images, CSS, JavaScript to add interactivity and increase ease of navigation.
    10. All the web pages should link together.
    11. All pages should be validated with no errors using https://validator.w3.org/
